# Security Policy

## Supported versions

Security fixes are handled on the current `main` branch unless maintainers publish a versioned support policy.

## Reporting a vulnerability

Please report suspected vulnerabilities privately through GitHub Security Advisories when available, or by contacting the maintainers through the project repository. Do not open a public issue for exploitable behavior.

Include:

- affected version or commit
- operating system and runtime
- reproduction steps
- expected impact
- any logs or packet traces with secrets removed

## Handling secrets

Never include real upstream proxy credentials, PAC URLs with secrets, private DNS logs, or captured traffic payloads in issues, logs, screenshots, or tests.
